Introduction

Natural Language Processing (NLP) is a rapidly growing area using computer science, mathematics and linguistics to handle text data. NLP is very important especially with processing dozens of text sources on the internet and to help people interact with computers.

The latter is the main topic of this project. The goal is to help the users to write computer texts with predictive text mining application, more precisely, to create a Shiny app that takes as input a phrase (multiple words) and predicts the next word.

This article is about introductory exploratory analysis and strategy hints.

Note: This is a technical version of the text with all R code. For the version without code chunks see Milestone Report - Manager Version.

Sentences

First of all, the relevant data structure for text prediction have to be chosen. The lines of data do not represent any meaningful structure. There are blog parts in the first data set, news in the second data set and tweets in the third data set. They do not correspond to each other.

At this point the unified approach has to be stated. It is not reasonable to have a long series of words for next word prediction. The hypothesis is, that only from two to five word segments should be appropriate. That is why shorter text structure shall be used. The sentence seems to be the good one.

Another reason for transforming all data to sentences is that the words order differs with respect to the basic sentence type (declarative, interrogative, and imperative). The distribution of these types should be studied and reflected.

    #http://www.cookbook-r.com/Graphs/Multiple_graphs_on_one_page_(ggplot2)/

    if (is.null(layout)) {
      # Make the panel
      # ncol: Number of columns of plots
      # nrow: Number of rows needed, calculated from # of cols

    if (numPlots==1) {

Conclusions:

  • The data will be transformed to sentences.
  • Only sentences ended with “.”, “!”, and “?” will be used for further analysis.
  • The length of sentences used for prediction will be limited to 200 symbols.
  • The prediction will be tested for the sentence types “.”, “!”, and “?” together and separately.

Words

As was mentioned before, the goal is to predict next word based on several forgone words, and the base data structure for word sequences will be sentence.

At this point it is necessary to do some exploratory analysis of the words and their distribution. Following setting is used for words creation:

  • conversion to lower,
  • removing number,
  • removing punctuation,
  • removing separators,
  • removing twitter specific symbols,
  • removing hyphens,
  • NOT removing stopwords.

The same setting will be used for pre-processing the given words in prediction.
